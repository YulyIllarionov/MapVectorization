1. Исходные данные:
   Отсканированная бумажная навигационная карта. Содержит:
   - закрашенные области с черной окантовкой и без. Области могут стыковаться между собой
   - линии разного цвета и стиля ( непрерывные, пунктирные и т.п. )
   - точки с надписями и без
   - тексты
   - знаки условных обозначений

   Размер растра карты до 20000*20000 пикселей
   Карта может быть частично недостаточно четкой и содержать артефакты, связанные с некачественным сканированием и неправильным хранением ( замятости, следы сгибов и т.п.)
   Пример двух карт по ссылке https://cloud.mail.ru/public/5K3w/xt3s7UW7f

2. Задача:
            Требуется разработать приложение распознавания и выдачи векторной информации по растровым данным карты. Требуется векторизовать объекты следующих типов ( в порядке убывания приоритета ):
            - замкнутые полигоны
            - непрерывные линии глубины
            - точки глубин
            - точки высот
            - значения точечных глубин из текста надписей
            - значения точечных высот из текста надписей
            - значения глубины для линий глубины
            - остальные сплошные линии
            - линии с прерывистыми стилями
            - тексты ( в текст с размером фонта )

   Модуль векторизации должен уметь работать как в полностью автоматическом, так и в полуавтоматическом режиме, когда объект для векторизации выбирается пользователем.  Также в случае сбоя векторизации ( не знает, по какой ветке идти) пользователь должен иметь возможность указать дальнейшее направление восстановления векторных данных.

Выходной формат - текстовый (SVG, DXF, XML). По усмотрению.

3. Требования к коду, архитектуре и составу:
     - C++, MSVS
     - Исполняемое приложение
     - Можно использовать готовые OpenSource библиотеки
